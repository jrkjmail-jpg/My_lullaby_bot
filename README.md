# My_lullaby_bot

## Переменные окружения

Для работы бота нужны:

```env
TELEGRAM_TOKEN=
OPENAI_API_KEY=
SUNO_API_KEY=
YOOKASSA_SHOP_ID=
YOOKASSA_SECRET_KEY=
YOOKASSA_RETURN_URL=https://t.me/username_бота
YOOKASSA_VAT_CODE=1
# YOOKASSA_TAX_SYSTEM_CODE=
YOOKASSA_WEBHOOK_HOST=0.0.0.0
YOOKASSA_WEBHOOK_PORT=3000
YOOKASSA_WEBHOOK_PATH=/yookassa-webhook
YOOKASSA_TEST_MODE=1
ADMIN_IDS=
DB_PATH=/app/shared/kolybelka.db
# PERSISTENCE_PATH=/path/to/persistent/kolybelka_state.pkl
TEXT_GENERATION_TIMEOUT_SECONDS=300
MUSIC_GENERATION_TIMEOUT_SECONDS=900
REMINDERS_ENABLED=1
REMINDER_AFTER_DAYS=14
REMINDER_INTERVAL_HOURS=24
```

`YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY` берутся в личном кабинете ЮKassa.
`YOOKASSA_RETURN_URL` можно указать ссылкой на Telegram-бота.
В личном кабинете ЮKassa нужно добавить webhook на событие `payment.succeeded`.
URL вебхука должен вести на публичный HTTPS-адрес сервера и путь из `YOOKASSA_WEBHOOK_PATH`, например `https://example.com/yookassa-webhook`.
На время модерации ЮKassa бот может запускаться без `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY`; в этом случае покупка будет временно недоступна, но остальные сценарии бота работают.
Для пробной оплаты используйте тестовый магазин ЮKassa и его тестовые `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY`. `YOOKASSA_TEST_MODE=1` только добавляет в сообщениях пометку, что оплата тестовая; API ЮKassa остаётся тем же.
`ADMIN_IDS` — список Telegram ID администраторов через запятую. Администраторы могут вручную начислять орешки командой `/addnuts`.
`DB_PATH=/app/shared/kolybelka.db` — важная настройка для BotHost. Перед деплоем включите у бота общее хранилище и задайте этот путь, иначе SQLite-база может оказаться в папке проекта, а при обновлении с Git или перезапуске орешки будут выглядеть как сброшенные. Если `DB_PATH` не указан, бот сначала использует `$SHARED_DIR/kolybelka.db`, затем `/app/shared/kolybelka.db` в среде `/app`, и только потом файл рядом с `bot.py`. При первом запуске с постоянным хранилищем бот попробует скопировать старую базу `kolybelka.db` из папки проекта, если в новом месте базы ещё нет.
`PERSISTENCE_PATH` можно не указывать: состояние диалогов хранится рядом с SQLite-базой в `kolybelka_state.pkl`. Это помогает пользователям не застревать на старых кнопках после перезапуска бота.
`TEXT_GENERATION_TIMEOUT_SECONDS` — критическое ожидание создания/правки текста, по умолчанию 5 минут.
`MUSIC_GENERATION_TIMEOUT_SECONDS` — критическое ожидание создания музыки, по умолчанию 15 минут.
`REMINDERS_ENABLED`, `REMINDER_AFTER_DAYS` и `REMINDER_INTERVAL_HOURS` управляют мягкими автонапоминаниями пользователям. По умолчанию бот проверяет напоминания раз в 24 часа и пишет тем, кто не создавал колыбельную 14 дней.

## Оплата

Бот продаёт орешки:

- 1 орешек — 349 ₽
- 2 орешка — 499 ₽
- 3 орешка — 599 ₽

Перед созданием платежа бот спрашивает электронную почту для отправки чека.
После оплаты бот получает webhook от ЮKassa, перепроверяет платёж через API и автоматически начисляет орешки.
1 орешек списывается только после того, как готовая музыкальная колыбельная отправлена пользователю.

## Навигация во время создания

Кнопка `👤 Личный кабинет` работает на этапах создания колыбельной как временная вкладка. Пользователь может открыть баланс, перейти к покупке орешков и нажать `⬅️ Вернуться назад`, чтобы попасть на тот же шаг создания, с которого он открыл личный кабинет. В таком режиме в личном кабинете нет кнопки `🔄 Начать заново`, чтобы случайно не сбросить текущий сценарий.

## Команды бота

### Пользовательские команды

- `/start` — открыть главное меню и начать работу с ботом заново.
- `/balance` — показать текущий баланс орешков и открыть личный кабинет.
- `/myid` — показать Telegram ID пользователя. Нужен, чтобы добавить администратора в `ADMIN_IDS`.
- `/stopreminders` — отключить мягкие напоминания о создании новых колыбельных.
- `/startreminders` — снова включить мягкие напоминания.
- `/cancel` — отменить текущий сценарий создания колыбельной и вернуться к главному меню.

### Админские команды

Админские команды работают только для Telegram ID из переменной `ADMIN_IDS`.

- `/addnuts user_id количество` — вручную начислить орешки пользователю. Пример: `/addnuts 123456789 1`.
- `/removenuts user_id количество` — вручную списать орешки у пользователя, если на балансе достаточно орешков. Пример: `/removenuts 123456789 1`.
- `/broadcast текст` — отправить произвольную рассылку всем пользователям. Пример: `/broadcast Сегодня вечером будет технический перерыв`.
- `/maintenance текст` — отправить всем пользователям готовое уведомление о техническом перерыве; текст после команды добавится к шаблону. Пример: `/maintenance Ориентировочно до 22:00 по Москве`.
- `/remindnow` — вручную отправить мягкое напоминание тем, кто давно не создавал колыбельную и не отключил напоминания.
- `/remindpreview` — показать администратору все варианты мягких напоминаний без рассылки пользователям.
- `/users` — показать, сколько пользователей есть в текущей базе, сколько подходят для мягкого напоминания, какой SQLite-файл использует бот, видит ли он старую базу рядом с `bot.py`, и последние ID/username пользователей. Полезно проверить перед `/broadcast` и `/remindnow`.
- `/dbstatus` — показать техническую диагностику SQLite: текущий путь базы, размер файла, количество пользователей, платежей и общее число орешков на балансах. Полезно, если кажется, что баланс сбросился.

Важно: рассылки отправляются только пользователям, которые уже есть в SQLite-базе `users`. Бот записывает пользователя при любом сообщении, а при запуске пытается объединить старую базу рядом с `bot.py` с текущей постоянной базой. Если `/users` показывает `0`, команда рассылки выполнится, но отправлять сообщение будет некому. Также убедитесь, что ваш Telegram ID добавлен в `ADMIN_IDS`, иначе админские команды будут отклоняться.

Мягкие напоминания не продают колыбельную напрямую. Они аккуратно подсказывают, что песню можно создавать не один раз, а под разные моменты жизни ребёнка: день рождения, первый день в школе, новую игрушку, поездку, новый велосипед, появление братика или сестрёнки, хороший день дома или спокойное завершение вечера.

## Пробная оплата ЮKassa

1. В личном кабинете ЮKassa создайте или откройте тестовый магазин.
2. Скопируйте тестовые `shopId` и `secretKey` в переменные `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY`.
3. Укажите `YOOKASSA_TEST_MODE=1`.
4. В настройках тестового магазина добавьте webhook на публичный адрес бота: `https://ваш-домен/yookassa-webhook`.
5. Включите событие `payment.succeeded`. При желании можно добавить `payment.canceled`.
6. Перезапустите бота на хостинге.
7. В боте нажмите `Купить орешки`, выберите количество, введите email и оплатите тестовой картой ЮKassa.
8. Для успешного теста можно использовать карту `5555 5555 5555 4477`, срок действия в будущем, CVC и код подтверждения — любые числа.
